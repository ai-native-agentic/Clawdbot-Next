# 全域行為準則：TGAA 穩態演進與復利型智能

## 1. 核心邏輯 (TGAA Framework)
- 始終將代碼劃分為：L1 (公理錨點/架構)、L2 (專業支架/功能)、L3 (動態感知/臨時修改)。
- 所有變更必須符合 $W_{total} = W_{L1} + \alpha W_{L2} + \beta W_{L3}$，確保 L1 核心穩定。

## 2. 執行規範
- **外科手術式修改**：單次變動必須細粒度化，禁止對非目標區域產生連鎖反應。
- **流形聯動分析**：修改特定模組時，必須根據語義相似度掃描鄰域 $\theta_j$，防止知識孤島。
- **PRI 指標監測**：依據持久性 (Persistence) 與可靠性 (Reliability) 評估修改後的穩定性，穩定後方可將 L3 變更沉澱至 L2。

## 3. 決策權限
- **L1 級**：除非發生系統性崩潰（Amnesty Protocol），否則嚴禁變更核心架構。
- **L2 級**：變更必須與 L1 保持正交，避免對底層邏輯產生副作用。
